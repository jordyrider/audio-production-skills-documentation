  <action>
Create `schemas/` at project root. Write five JSON schemas for the content file types.

All schemas share a common pattern:
- `$schema`: "http://json-schema.org/draft-07/schema#"
- `title`: descriptive name
- `type`: "object"
- `required`: always includes `["type", "status"]`
- `properties`: field definitions
- `additionalProperties`: false

**Status and type enums** (identical across all content schemas):
```json
"status": {
  "type": "string",
  "enum": ["stub", "draft", "validated"]
},
"type": {
  "type": "string",
  "enum": ["genre", "supergenre", "tool", "skill", "voice-category"]
}
```

**Stub exception pattern** — use `if/then/else` or `oneOf` to enforce:
- When `status` = "stub": content fields accept `null` OR the expected array/object type
- When `status` = "draft" or "validated": content fields must be non-null and non-empty

Implement the stub exception using `if/then/else` at the schema root:
```json
{
  "if": { "properties": { "status": { "const": "stub" } } },
  "then": { /* all content fields nullable */ },
  "else": { /* all content fields required non-null */ }
}
```

  <action>
Create `schemas/index.schema.json` for per-folder YAML index files.

Index files are folder registries — they list the files and sub-folders within their folder. The genre index is special: it is the authoritative source of genre IDs referenced by tool files.

**index.schema.json** structure:
```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "Folder Index",
  "description": "Per-folder registry of files and sub-folders in the skills knowledge base",
  "type": "object",
  "required": ["folder", "type", "files"],
  "properties": {
    "folder": {
      "type": "string",
      "description": "Relative path of this folder from project root"
    },
    "type": {
      "type": "string",
      "enum": ["genre-index", "music-index", "voice-index", "tools-index", "skills-index", "subgenre-index"],
      "description": "What kind of index this is"
    },
    "files": {
      "type": "array",
      "description": "Files in this folder",
      "items": {
        "type": "object",
        "required": ["id", "file", "type", "status"],
        "properties": {
          "id": { "type": "string", "description": "Stable identifier for cross-referencing (e.g. 'lo-fi', 'music-2.5')" },
          "file": { "type": "string", "description": "Filename (not full path)" },
          "type": { "type": "string" },
          "status": { "type": "string", "enum": ["stub", "draft", "validated"] },
          "title": { "type": "string" },
          "description": { "type": "string" }
        },
        "additionalProperties": false
      }
    },
    "sub_folders": {
      "type": "array",
      "description": "Sub-folders within this folder (optional)",
      "items": {
        "type": "object",
        "required": ["id", "folder"],
        "properties": {
          "id": { "type": "string" },
          "folder": { "type": "string" },
          "title": { "type": "string" }
        },
        "additionalProperties": false
      }
    }
  },
  "additionalProperties": false
}
```

The `id` field in file entries is the stable identifier used for cross-referencing. For genre files, these IDs are the authoritative genre IDs. Tool indexes reference these IDs in stable_styles entries.
